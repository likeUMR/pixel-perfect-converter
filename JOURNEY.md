# 探索历程：从零到 Pixel Perfect

## 起点：问题定义

AI 大模型可以生成像素风格图片，但这类输出并非真正的像素艺术。原因在于：

- 每个"逻辑像素"在图像中对应若干物理像素（比如 8×8 像素的格子），但格子内的颜色并不均匀——有抗锯齿，有渐变
- 更关键的是，AI 对像素网格的理解并不稳定：在图像的一个区域对齐得很好，在另一个区域可能整体偏移了几个像素（网格漂移）

目标：给定一张 AI 生成的白底像素风格图，自动检测其逻辑像素大小，找到最优网格，将每个格子恢复成单一纯色，生成真正 pixel perfect 的版本。

---

## 第一轮：直接上手（`pixel_perfect_checker.py`）

第一个版本的思路最为直接：

1. 去掉白色背景（BFS 泛洪）
2. 用梯度分析估计像素大小
3. 对每个格子取平均色
4. 输出修复后的图

**观察**：效果不佳。核心问题是网格漂移——用单一全局网格对齐，在图像边缘区域会错位，导致修复后的像素位置与原始像素不对应，产生明显的扭曲。

---

## 第二轮：三种算法并行探索

意识到问题的核心是"网格并非全局统一"，于是设计了三种不同的思路分别实现。

### 算法 1：撒豆算法（Bean Seed）

**核心思路**：不假设全局网格，而是在图像上密集撒点（种子），每个种子独立找到自己所在的最大纯色正方块，然后把所有方块聚合起来，推断像素大小，再进行网格对齐。

**完整流程**（每步保存中间结果）：
1. 去白底
2. 以约 3px 步长在内容区域撒入大量种子
3. 每个种子：在预计算的各尺度方差图上，找到**包含该种子的最大纯色正方块**
4. 合并重叠/重复的方块（保留方差最低的）
5. 从方块尺寸分布推断像素大小（去掉离群值后取中位数）
6. 将所有非离群方块修正到 `pixel_size × pixel_size`，填充为纯色
7. 将过大的离群方块分裂成多个子像素
8. 全局网格对齐（直方图众数法找最优 offset）
9. 对空格子用邻居颜色填充
10. 最终输出

**迭代改进**：

- 初版正方块查找是以种子为中心的。改进后变为"**找包含种子的最大纯色正方块**"——种子不必在中心，只要在方块内即可。这样对边角区域的种子效果更好。
- 提取了 `config.py` 配置文件，把纯色阈值、合并阈值等所有参数统一管理
- 实现了 `tune.py` 调参扫描工具，用多组参数组合批量生成对比图

**优点**：对局部漂移有一定鲁棒性，因为每个像素是独立识别的。
**缺点**：对于大块同色区域（没有任何内部边界信号），种子很难找到正确的方块边界；精细结构容易发生轻微扭曲。

### 算法 2：分块网格对齐（Block Grid）

**核心思路**：把图像切分成若干大块（如 3×3、4×4 等多种切分策略），在每块内独立做梯度频谱分析，得到局部的像素大小和网格 offset。比较各块的一致性，评分最高的切分策略获胜。

**优点**：能直接可视化网格漂移——不同块之间的 offset 差就是漂移量。
**缺点**：拼接边界处理复杂，综合质量不如算法 1。

### 算法 3：梯度 + Hough 网格拟合

**核心思路**：
- 计算 Sobel 梯度图（像素边界处梯度最强）
- 将梯度投影到水平/垂直轴，做 FFT 频谱分析，找到主频（= 像素大小）
- 相位对应最优 offset
- 在不同区域滑窗分析，得到局部漂移场

**优点**：对"哪里漂移、漂多少"分析最清晰。
**缺点**：大片同色区域完全没有梯度信号，频谱峰值不够尖锐，检测精度受限。

### 三种算法的对比观察

锁定 `pixel_size=8` 后，对 `pictures/` 目录下多张图片并行对比：

- 撒豆算法整体视觉效果最接近原始图
- 分块算法在内容均匀的图上表现稳定，但边界衔接有割裂感
- 梯度算法的漂移分析最精确，但修复质量不稳定

**关键观察**：有些 AI 生成的图，本身已经非常接近 pixel perfect（特别是在传入了参考网格图的情况下）。对于这类图，任何"修复"算法都会引入不必要的失真——最好的做法是先**评估**遵循度，如果已经足够好，直接用简单的网格采样修复即可。

## 转折点：重新定义问题

观察到一个重要现象：当给 AI 传入参考网格图（如 `grid_1px_black.png`）作为 ControlNet 输入时，AI 在多数情况下会主动遵循网格，生成的图非常接近 pixel perfect。偶尔才会"不听话"，出现漂移。

这意味着：**问题的本质不是"如何修复复杂的网格漂移"，而是"如何快速判断 AI 是否遵循了网格，并在此基础上做最小侵入的修复"**。

原有的三种算法都是针对"严重漂移"设计的，逻辑复杂、对轻微漂移反而会过度干预。

## 算法 4：经典网格 + 遵循度评估

### 核心思路

**假设网格是已知的**（从参考图解析出 `pixel_size=8, offset=(0,0)`），在此基础上：

1. **评估**：分析每个 8×8 格子的颜色纯度、格间边界锐利度，给出遵循度评分
2. **修复**：用格子中心 4×4 区域的中位色直接填满整个 8×8 格子

### 为什么是 4×4 而不是 8×8？

AI 生成的图即使遵循了网格，格子**边缘**依然存在抗锯齿（与相邻格子颜色的混合）。如果取整个 8×8 格子的平均色，会被边缘混色污染。取中心 4×4 的中位色，避开了边缘干扰，得到更纯净的"格子真实颜色"。

如果 AI 还在图中保留了 1px 的网格线，则对 `grid_margin=1` 进行额外裁剪，方差计算从 6×6 内部区域进行，更鲁棒。

### 遵循度指标设计

| 指标 | 计算方式 | 意义 |
|------|----------|------|
| `CPR_hard` | 格内方差 < 200 的格子比例 | 严格纯色率 |
| `CPR_soft` | 格内方差 < 2000 的格子比例 | 宽松纯色率 |
| `MCV` | 内容格子的平均方差 | 整体颜色均匀性 |
| `BIR` | 格间颜色差 / 格内颜色标准差 | 边界锐利度 |
| `score` | `CPR_soft×40 + min(BIR/6,1)×60` | 综合评分 |

**BIR（边界/内部比）** 是最核心的指标：在真正 pixel perfect 的图中，格子**之间**的颜色差异应该远大于格子**内部**的颜色差异。如果 AI 的输出颜色连续渐变，BIR 就会很低；如果严格按格子填色，BIR 就会很高。

### 后处理

- **透明背景**：对像素完美输出做格子级 BFS，从四边白色格向内扩散，识别背景并置为透明，输出 RGBA PNG
- **迷你版**：每个逻辑像素（8×8 的格子）缩成 1 个像素，适合游戏 sprite 使用

### 调试中遇到的 Bug

**Bug 1：像素完美输出还有网格线**

初版 `render_pixel_perfect` 代码中，在填充格子颜色后，又显式绘制了 1px 黑色网格线。这与"纯色块"的目标矛盾。修复：删除网格线绘制代码，直接用 `cell_y0:cell_y1` 覆盖整个格子（包括原来的网格线位置）。

**Bug 2：透明背景对满屏角色无效**

初版透明背景依赖 `cells.is_bg` 标记，而这个标记来自对原始图的 BFS 去白底。当角色铺满整个画布时，白色背景被角色遮挡，原始图 BFS 无法正确识别背景区域，导致所有格子都被标为"内容格"。

修复：在像素完美输出（已是干净的纯色块）上重新做一次格子级 BFS。从四边的白色格出发向内扩散，与原始图完全解耦，更稳定。

## 为什么选择"Mod 模式"而不是微调/训练模型

这个问题值得单独说清楚。

### Mod 模式的优势

"Mod 模式"指的是：不改动 AI 模型本身，而是通过**预处理**（精心设计的参考图、Prompt）和**后处理**（网格遵循度评估 + 像素修复）来弥合 AI 输出与目标格式之间的差距。

**成本极低**：不需要准备训练数据，不需要 GPU 时间，不需要任何模型训练基础设施。整个后处理 pipeline 在 CPU 上几秒内完成。

**效果上限直接继承大模型**：AI 的生成能力（理解语义、构图、色彩搭配、风格迁移）完全保留。我们只是在输出上做一次"格式化"，而不是改变 AI 的能力。

**模型无关**：今天用 Flux，明天换成更强的 GPT-5 或其他模型，后处理 pipeline 完全不需要改动，立刻能用上更强的模型。

**灵活可调**：格子大小、采样策略、纯度阈值都是可配置的参数，可以针对不同风格的像素艺术快速调整，不需要重新训练。

### 微调/训练的局限性

**小规模训练效果天花板低**：直接微调一个像素风格输出的模型，需要大量高质量的（输入, pixel-perfect 输出）配对数据。小规模数据集微调的效果远不如大厂在海量数据上预训练的大模型。从一开始，"能力上限"就被压低了。

**维护成本高**：一旦出了更好的基础模型，微调模型需要重新训练。这是持续性的时间和计算成本。

**泛化能力弱**：对特定角色、特定风格微调的模型，在遇到分布外的内容时容易崩坏，而大模型的泛化能力要强得多。

### 结论

对于像素艺术后处理这类"格式规范明确"的任务，Mod 模式是显著优于微调的策略。大模型负责"想"（创意、语义理解），后处理 pipeline 负责"整理"（格式合规）。两者各司其职，效果最优，成本最低。

## 最终工作流总结

```
1. 准备参考网格图 pixel_grid/grid_1px_black.png（8×8 网格，512×512）

2. 用 nanobanana2（或任意 AI 工具）生成：
   Prompt: "生成一个白色背景像素风格的[描述]（参考图片对像素进行填充，保留网格）"
   参考图: grid_1px_black.png

3. 用 algo4/evaluate.py 批量评估：
   python algo4/evaluate.py --src pictures/ --grid-ref pixel_grid/grid_1px_black.png --pixel-size 8

4. 查看 report_card.png 和 summary.txt，筛选高分图

5. 使用输出：
   pixel_perfect_transparent.png  → 游戏角色/合成用
   pixel_mini.png                 → 游戏 sprite 用
```
