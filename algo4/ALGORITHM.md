# Algo4：经典网格识别 + 网格遵循度评估

## 背景与动机

在 AI 绘制像素风格图片时，通常会给 AI 提供一张参考网格图（如 `pixel_grid/grid_1px_black.png`），要求 AI 在网格内绘制，使每个逻辑像素（pixel）严格对应一个网格格子。

现实中 AI 的表现差异很大：
- **遵循好**：每个格子内颜色高度一致，颜色切换恰好在格线处发生（如骑士图）
- **遵循差**：格内颜色混乱，边界模糊，实际内容跨越格线（如另一骑士图）

Algo4 的目标是：
1. 从参考网格图中自动提取网格参数（pixel_size 和 offset）
2. 判断 AI 的输出图是否包含可见网格线
3. 逐格分析每个格子的颜色纯净度
4. 用三项指标量化 AI 的遵循程度，综合为一个 0-100 的"遵循分"
5. 输出像素完美版本（经典网格法），并生成可视化对比

---

## 步骤详解

### 步骤 1：从参考网格图解析网格参数

**输入**：参考网格图（`grid_1px_black.png`，512×512）

**处理**：
1. 读取图像为 NumPy 数组
2. 对每一列，计算该列所有像素的平均 RGB 值（即该列的"平均亮度"）
3. 找出平均亮度低于阈值（默认 80）的列 → 这些列是黑色网格线
4. 用相邻黑色列的间距众数 → 得到 `pixel_size`
5. 第一条黑线的位置 → 得到 `offset_x`（对 Y 方向重复同样操作得到 `offset_y`）

**示例**：`grid_1px_black.png` 的黑线在列 0, 8, 16, 24, ...，所以 `pixel_size=8, offset=(0,0)`

---

### 步骤 2：检测图像中是否存在可见网格线

**输入**：AI 生成图像的 RGB 数组、步骤 1 得到的 pixel_size 和 offset

**处理**：
1. 对图像每个像素计算亮度（RGB 均值），低于 80 的认为是"暗像素"
2. 统计每列的暗像素比例 → `col_dark_ratio`
3. 将列按是否属于"期望的网格线位置"分为两组：
   - 网格线列：`x ≡ offset_x (mod pixel_size)`
   - 非网格线列：其余列
4. 计算两组暗像素比例的均值，取比值 `ratio = 网格线组 / 非网格线组`
5. 若 `ratio > 2.5`（无论水平方向还是垂直方向），判定"可见网格线存在"

**意义**：
- 如果 AI 将网格线保留在输出中，则每个格子的第 0 行和第 0 列是黑色 → 后续分析时需跳过这 1px 边距（`grid_margin=1`）
- 如果 AI 没有保留网格线（例如直接在空白画布上作画），则使用全格内容（`grid_margin=0`）

---

### 步骤 3：逐格分析

**输入**：RGB 数组、有效像素掩码（valid，去除白色背景后透明区域为 False）、网格参数

**处理**：

对网格内每个格子 `(gy, gx)`（格子坐标从 `offset` 出发，间距为 `pixel_size`）：

1. 确定格子在图像中的内容区域（排除 `grid_margin` 边距）：
   ```
   内容区域 y ∈ [offset_y + gy*ps + gm, offset_y + (gy+1)*ps - gm)
   内容区域 x ∈ [offset_x + gx*ps + gm, offset_x + (gx+1)*ps - gm)
   ```

2. 提取该区域的有效像素（`valid == True`）

3. 计算统计量：
   - **中位色**：对所有有效像素的 R/G/B 分别取中位数 → `median_color`（比均值更鲁棒）
   - **格内方差**：所有有效像素（N×3 矩阵）的3个通道方差之和 → `variance`
   - **覆盖率**：有效像素数量 / 总像素数量 → `coverage`

4. **背景格判定**：
   - 若 `coverage < 0.3`（格内有效内容不足 30%），或格内均值亮度 > 230 → 标记为背景格，不参与指标计算

**输出**：一个以 `(gy, gx)` 为键的字典，每项包含格子的所有统计信息

---

### 步骤 4：计算遵循度指标

**输入**：步骤 3 的格子字典（仅内容格参与计算）

#### 指标一：格子纯色率（CPR, Cell Purity Rate）

对所有内容格，统计方差低于阈值的比例：

- `CPR_hard`（严格纯色率）：格内方差 < 200 的格子占比
- `CPR_soft`（宽松纯色率）：格内方差 < 2000 的格子占比

> **直觉**：一个严格遵循网格的格子，内部所有像素颜色几乎完全相同，方差接近 0。AI 的略微抗锯齿或颜色噪声会使方差略高，因此设两档阈值。

#### 指标二：格内平均方差（MCV, Mean Cell Variance）

```
MCV = mean(variance for all content cells)
```

越低越好。完美遵循时，MCV 接近 0；完全不遵循时，MCV 可达数万（相当于 RGB 各通道标准差超过 100）。

#### 指标三：边界/内部比（BIR, Boundary-to-Interior Ratio）

```
within_std  = mean(sqrt(variance_i) for all content cells)       ← 格内色彩波动
between_diff = mean(|median_i - median_j|₁ for all adjacent cell pairs)  ← 格间色差

BIR = between_diff / (within_std + ε)
```

**直觉**：
- 遵循好的图像：每格内部高度一致（`within_std` 小），相邻格颜色可以差距大（`between_diff` 大） → BIR 高
- 遵循差的图像：格内颜色混乱（`within_std` 大），格间差距反而不大（因为颜色在格内已经充分混合） → BIR 低

示例验证：
| 图像 | BIR |
|------|-----|
| 好遵循（骑士A） | 3.86 |
| 差遵循（骑士B） | 0.44 |

#### 综合分

```
score = CPR_soft × 40 + min(BIR / 6, 1) × 60
```

- CPR_soft 最高贡献 40 分（格子纯色率）
- BIR ≥ 6 时满分 60 分（边界锐利）
- 满分 100 分

---

### 步骤 5：像素完美渲染

经典网格法：

1. 创建白色输出图
2. 若有可见网格线：在每格的左边列和上边行画黑色（还原网格线）
3. 对每个内容格：用其 `median_color` 填充格内所有内容像素
4. 对背景格：保持白色

这是最简单的像素完美修复，不涉及任何方块搜索，直接按网格分格取颜色。

---

### 步骤 6：可视化

#### 纯色热力图
- 深绿：方差 < 200（严格纯色）
- 黄绿：方差 < 2000（宽松纯色）
- 橙：方差 < 5000（中度混浊）
- 深红：方差 ≥ 5000（高度混浊）
- 灰：背景格

#### 边界锐度图
对每对相邻内容格，在它们之间的边界线上绘制色差强度：
- 绿（色差 > 80）：边界锐利 → 遵循好
- 黄（色差 30-80）：中等
- 红（色差 < 30）：边界模糊 → 遵循差

底图用各格中位色的 50% 混入深色，让位置关系更清晰。

## 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `PURITY_THRESH_HARD` | 200 | 严格纯色阈值（3通道方差和） |
| `PURITY_THRESH_SOFT` | 2000 | 宽松纯色阈值 |
| `BG_THRESH` | 230 | 背景格亮度阈值（均值高于此视为背景） |
| `DARK_LINE_THRESH` | 80 | 暗像素阈值（均值 RGB） |
| `GRID_LINE_COL_RATIO` | 0.25 | 列中暗像素占比阈值（网格线检测） |
| `BIR_CAP` | 6.0 | BIR 归一化上限（BIR ≥ 此值得满分） |

## 已知局限

1. **偏移假设**：代码假设已知 `pixel_size` 和 `offset`（从参考网格解析），若 AI 输出的比例与参考网格不一致，需手动指定 `--pixel-size`。
2. **背景格判定**：依赖亮度阈值，对颜色较浅的内容格（如淡黄色格子）可能误判为背景。
3. **网格线可见检测**：依赖暗像素密度的比较，若角色大量使用黑色可能影响判定。
4. **单一全局网格**：不支持局部网格漂移；若 AI 的网格在边缘发生位移，指标会偏低但难以定位具体位置（边界锐度图有一定的定位能力）。
